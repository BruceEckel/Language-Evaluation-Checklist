Language Evaluation Checklist
To Compare the Experience of Adopting Different Programming Languages
General Intent/Goals Functional + OO on the JVM
Competitors Clojure, Ceylon, Kotlin
Issues & Constraints Slow compile times; Standard library compatibility; Not FP enough for Haskell crowd; Significantly variety in language use;
Ease of Use
Complexity Some great simplifications over Java: No primitives, First-class functions, Type inference Some really complex stuff: Implicits, In-fix notation, Curried functions, Monads
Learning Curve Heavy
Install Process REPL / scalac or Standard build tool (sbt)
External Library Install Process Maven & Ivy repo support via sbt (or other build tools)
Learning Tools Official Tutorial
Active Beginner Community
Old-style Newsgroup
Multiple (Confusing) Newsgroups
StackOverflow Group
Popularity
(Influences ease of learning/hiring) 		Right up there with R, Perl, and Shell: Github
Redmonk
Tiobe
IEEE
PYPL
Community & Culture
Focus Academic, Enterprise, Big Data
Code of Conduct Retroactive due to bad behavior
Conference Central
Satellites
Attitude Toward Beginners Depends on the sub-community. The Play and Spark communities are pretty friendly to noobs.
Maturity Indicators StackOverflow Tag
Ability to Fix Language Mistakes Happens but difficult to balance with those that don't want upgrade pain.
Tools
Style Guide and level of enforcement Exists but Optional
Build sbt is the standard but others can be used. Understanding sbt is very difficult resulting in generally low satisfaction.
Lint Exists but not default
Test Framework ScalaTest & Specs2 are most used but any JVM / JUnit test framework will work.
Add-On Libraries
Distribution System Maven Central is the primary but any Maven-style repo works.
Installation Process Depends... Activator / sbt is a simple download, unzip, and run process.
Installation tool Part of sbt.
Semantic Versioning
(For libraries as well as language itself) 		Anything goes
Governance
IP ownership EPFL
License 3-clause BSD license
Funding EPFL & Typesafe
Process to suggest new features GitHub & Scala Improvement Process
Design Comes Primarily From Typesafe & EPFL
Pull Request Validation System Exists
Contributor's License Agreement (CLA) Required
Release Rate, Open/Closed Bugs
(indication of activeness) 		Good
*The GPL is sometimes used as a way to force companies to buy licenses.

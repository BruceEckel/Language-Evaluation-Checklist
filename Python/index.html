Language Evaluation: Python
To Compare the Experience of Adopting Different Programming Languages
General Intent/Goals Optimize programmer experience and productivity.
Competitors Ruby has some similarities but is not really a competitor.
Issues & Constraints Cannot support native parallelism. (Supports native asynchronous programming. Supports parallelism with multiprocessing library).
Speedups sometimes require writing components in alternate language (C, Pyrex, etc.).
Static type checking is limited and optional (only available in Python 3.4 & 3.5).

Complexity Focuses on simplicity and clarity above all.

Tools
Style Guide and level of enforcement Exists but Optional; tools available to enforce.
Build Dynamic language; no build tool necessary.
Lint Multiple linters are available.
Test Framework Multiple test frameworks available; py.test seems to be the current favorite.
Add-On Libraries
Distribution System Centralized with hosted repository, although you can also install directly from a Github repository with a single command. Virtual environments supported directly, to compensate for language and library version differences.
Installation Process Single Command: pip install library-name
Installation tool pip installed automatically with Python installation.
Semantic Versioning
(For libraries as well as language itself) 		Support is available but nothing seems specified or enforced.
